Basic Usage

# Run comprehensive evaluation on all theories. This command initiates a full simulation and validation pipeline across all implemented gravitational theories, computing particle trajectories using geodesic equations, checking conservation laws like energy and angular momentum preservation, and verifying against experimental datasets such as Mercury's perihelion precession and gravitational lensing observations. It ensures numerical stability through adaptive timestepping and provides detailed reports on theory performance, including precision metrics and failure modes, enabling rigorous comparison of theoretical predictions with empirical data.
albert run
# Include candidate theories in the evaluation. This flag extends the simulation to encompass experimental or hypothetical theories in the 'candidates' directory, allowing for systematic testing of novel modifications to established frameworks like general relativity, such as asymmetric metrics or quantum-corrected terms, while maintaining the same validation rigor including conservation checks and experimental matching.
albert run --candidates
# Test your own custom theory by adding it to the theories/ directory. This command filters the evaluation to focus exclusively on the specified theory, facilitating rapid iteration and debugging by isolating its performance in geodesic integration, conservation law adherence, and alignment with observational data, which is crucial for developing and refining new physical models.
albert run --theory-filter "MyTheory"
# Run simulations with extended trajectories by increasing the maximum number of integration steps. This allows for longer-term evolution of particle paths, which is essential for detecting subtle deviations in orbital stability, precession rates, or conservation violations that may only manifest over extended timescales in extreme gravitational regimes.
albert run --max-steps 10000
# Execute pre-flight tests to verify solver correctness before main simulations. These tests include analytical benchmarks like Keplerian orbits, numerical convergence checks, and validation against known solutions to ensure the integrator preserves symplectic structure and maintains required precision across various spacetime curvatures.
albert run --test

Available Commands

🚀 albert run

Run gravitational theory simulations with comprehensive validation. This core command orchestrates the entire pipeline: theory selection, geodesic equation solving using Runge-Kutta methods, trajectory generation, and multi-faceted validation including conservation laws and experimental data matching.

--theory-filter "name" # Select specific theories for focused evaluation, enabling targeted analysis of particular models without processing the entire suite, which is useful for iterative development and debugging of individual theories.
--steps 10000 # Set the number of integration steps for trajectory computation, allowing for finer temporal resolution in simulations to capture high-frequency phenomena or long-term stability characteristics.
--gpu-f32 / --cpu-f64 # Choose computation backend and precision: GPU with float32 for speed in large-scale simulations, or CPU with float64 for maximum numerical accuracy in precision-critical validations.
--enable-sweeps # Activate parameter sweep mode for systematic exploration of theory parameters, generating comprehensive datasets for analyzing sensitivity and optimizing model configurations against validation criteria.

🧪 albert discover

Generate new theoretical variations using AI-assisted code generation, creating novel modifications to base theories while ensuring they remain mathematically consistent and ready for immediate validation.

--initial "prompt" # Provide an initial description or concept to guide the generation of new theory variants, such as "add quantum correction term" or "modify for asymmetric metrics".
--from-theory path # Base the new theory on an existing implementation at the specified path, allowing for incremental modifications and extensions of proven models.

⚙️ albert setup

Configure API keys, computational resources, and network settings through an interactive wizard, ensuring optimal setup for simulations including GPU detection and precision settings.

Interactive wizard # Step-by-step configuration process that detects hardware capabilities, sets environment variables, and validates installations for seamless operation.

📊 albert validate

Perform standalone validation of a specific theory against the full suite of experimental datasets and conservation checks, generating detailed reports on performance metrics.

theories/my_theory/theory.py # Path to the theory implementation file to validate, allowing for isolated testing of individual models outside the full run pipeline.

🚀 albert run-advanced

Execute advanced simulations with parameter sweeps, parallel processing, and in-depth analysis tools for comprehensive theory exploration and optimization.

--enable-sweeps # Enable systematic parameter variation across specified ranges, generating multi-dimensional datasets for analyzing theory behavior and identifying optimal configurations.
--sweep-workers 8 # Set the number of parallel workers for distributed computation of parameter sweeps, leveraging multi-core or multi-GPU systems for accelerated processing of large simulation sets.

Use albert <command> --help for detailed options

How Solvers & Validators Work Together

The Albert framework operates on a fundamental principle: solvers generate physics, validators verify it. This separation of concerns ensures both computational efficiency and scientific rigor.

🚀 Solvers: The Physics Engines

Solvers are numerical integrators that evolve particle trajectories through spacetime by solving the geodesic equations. They handle:

  • ▸
    6D Phase Space Evolution: Position (r, θ, φ) and momentum (pr, pθ, pφ) components evolved simultaneously to preserve symplectic structure
  • ▸
    Adaptive Timesteps: Dynamic dt adjustment based on local curvature and velocity to maintain accuracy
  • ▸
    Theory-Specific Integration: Specialized solvers for UGM (tetrad formalism), quantum (path integrals), and extreme gravity regimes
  • ▸
    Runge-Kutta 4th Order: Default integration scheme with O(dt⁴) error scaling for stability

✓ Validators: The Reality Checks

Validators ensure theoretical predictions match experimental observations by testing against real-world data:

  • ▸
    Conservation Laws: Verify energy and angular momentum remain constant (within numerical tolerance) during evolution
  • ▸
    Experimental Matching: Compare predictions to datasets from Mercury's orbit, gravitational lensing, black hole shadows, etc.
  • ▸
    Precision Tolerance: Adaptive thresholds based on theory type and expected physical violations
  • ▸
    Multi-Scale Testing: From quantum interferometry (10⁻¹⁰ m) to cosmological distances (10²⁶ m)

The Feedback Loop:

  1. Theory Definition: User implements metric tensor g_μν(x)
  2. Solver Execution: Integrates geodesic equations to generate trajectories
  3. Validator Analysis: Checks conservation laws and experimental agreement
  4. Result Feedback: Pass/fail with detailed error metrics and suggestions
  5. Theory Refinement: Iterate based on validation results

A Validation Framework for Fundamental Physics

AlphaFold succeeded by creating specialized environments for training on protein structures. Albert provides analogous infrastructure for physics—leveraging PyTorch as the computational foundation. Express your theory as code—metric tensors and field equations—and Albert's solvers transform these mathematical laws into differentiable computations. This extends PyTorch into a platform for physics simulation, where Lagrangians can be implemented as differentiable operations yielding executable dynamics.

The differentiable nature enables theories to learn from their own simulations through gradient-based optimization. By providing this integrated infrastructure, Albert reduces the barrier to developing physics learning systems, allowing researchers to focus on innovation rather than building validation pipelines from scratch.
